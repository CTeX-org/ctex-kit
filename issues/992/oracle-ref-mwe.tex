% \verb* must scan the literal source at the call site; it cannot be passed
% through an ordinary macro argument.  \Compare prints the space label and
% starts the verbatim scanner, then \CompareResult measures and renders the
% oracle/candidate columns after the closing delimiter.
\newcommand\Compare[1]{%
  \texttt{#1}\quad\verb*}

\newcommand\CompareResult[2]{%
  \MatrixReset
  \global\setbox\OracleBox=\hbox{#1}%
  \MatrixReset
  \global\setbox\CandidateBox=\hbox{#2}%
  \global\BoundaryDelta=\dimexpr\wd\CandidateBox-\wd\OracleBox\relax
  \global\AbsoluteDelta=\BoundaryDelta
  \ifdim\AbsoluteDelta<0pt
    \global\AbsoluteDelta=-\AbsoluteDelta
  \fi
  &\fbox{\copy\OracleBox}&
  \fbox{\copy\CandidateBox}&
  \ifdim\AbsoluteDelta<0.01pt
    \textcolor{teal!70!black}{same}%
  \else
    \textcolor{red!80!black}{\the\BoundaryDelta}%
  \fi
  \tabularnewline
}

{\Large\bfseries xeCJK command-boundary oracle: kernel \texttt{\string\ref}}\par
\textbf{\EvidenceLabel}\par
\smallskip
Direct input is the oracle. The candidate must have the same visible content
and the same boundary spacing. The two-bit prefix records left/right source
spaces; \texttt{\string\verb*} scans the candidate source directly and marks
every literal space, without reconstructing that source through xeCJK.
For visibility this MWE uses
\texttt{CJKecglue=5pt} and \texttt{CJKglue=1pt}; the red value is
\(\mathrm{candidate}-\mathrm{oracle}\).
