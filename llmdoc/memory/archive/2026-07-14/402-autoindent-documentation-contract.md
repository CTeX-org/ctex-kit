# 反思：#402 autoindent 零缩进文档契约

## 把“未说明”与“未实现”分开

#402 延续自 #401 的讨论，但当前代码并不存在待修运行缺陷。真正缺口是用户手册把
`autoindent` 简写成“字号变化时自动调整”，没有说明 `\parindent=0pt` 会阻止更新。
因此本次修改不应删除条件判断或制造新的行为，而是把已有兼容语义提升为公开契约并
增加回归保护。

保留零值有两层理由：早期尚无 `autoindent=false` 时，旧文档只能用
`\parindent=0pt` 关闭自动调整；LaTeX 的 `minipage`、居中段落等结构又会主动使用
零缩进。若改成无条件赋值，就会同时破坏旧文档和结构内部版式。

## 文档型修复的视觉证据

运行行为没有变化时，不能用两张运行截图声称“修复前后”不同。本 PR 把证据拆成：

- 同页手册截图证明修改前遗漏例外、修改后写明行为和理由；
- 独立 MWE 同时展示非零缩进更新、显式零缩进保持和 `minipage` 保持，证明文档描述
  的既有运行契约。

这样既满足可见对比，也不会把文档修复包装成代码修复。
