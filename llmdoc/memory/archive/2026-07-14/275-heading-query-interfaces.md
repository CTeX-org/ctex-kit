# 反思：#275 标题查询接口

## 从历史 workaround 还原当前需求

#275 的原始代码属于 ctex 1.x 时代，不能直接把旧方案当成今天的接口设计。调查当前
Beamer 与 SJTUBeamer 后，真实缺口收敛为三类数据：裸编号、完整本地化标签和独立的
编号开关；标题文本与视觉样式已经由 Beamer 完整拥有。

这次接口之所以保持很小，是因为先用真实下游代码枚举私有依赖，再按语义去重，而不是
按六个私有变量逐个做公共别名。层级参数化同时覆盖 part、section、subsection，并为
其他 ctex 类复用同一模型。

## predicate 不是编号字符串的附属品

`numbering=false` 不只是“编号字符串为空”。主题通常还要移除编号旁的 stamp、换行和
间距；如果只提供编号和标签，主题仍需猜测状态或访问私有布尔值。独立 predicate 让
下游能删除整个附属布局，同时保持数据查询与排版决策分离。

## 两类视觉证据回答不同问题

自定义 MWE 证明新接口能产生过去做不到的正确变化：关闭编号后，标签和附属布局消失。
SJTUBeamer 的 9 页对比则证明迁移本身不改变既有视觉结果，逐页 `AE=0`。功能验证和
迁移无回归不能由同一张“看起来一样”的截图同时证明，应分别设计 oracle。

## 流程教训

- `scripts/check-parallel.sh` 只复制 `git ls-files` 可见路径；新测试仍完全未跟踪时，
  全量输出会继续显示旧的 183 项。确认新文件进入索引后才得到 184 项。
- `l3build check heading-query01 -q` 会把尾部 `-q` 当作测试名；选项必须位于测试名之前。
- 新增手册段落最初产生 26.6pt overfull。把一段职责过多的说明拆成短段后消除，说明
  API 文档也应把“返回什么”“动态语义”“不负责什么”分开表达。
- 第一轮自动审查的总评是 APPROVE，但表格仍列出 1 个小问题：实现注释未说明
  `\CTEXheadinglabel` 与 `\CTEXthe<level>` 的等价关系。初次收尾把它当作可选重复信息
  跳过，最终 completion audit 才按“所有大中小问题均处理”的原目标补上。审查总评
  不能覆盖详情中仍枚举的 finding；完成判定应逐项对照发现数量，而不是只读结论标签。
